# Create PR Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a reusable `create-pr` skill that infers the base branch when possible, folds clearly related uncommitted work into the PR when appropriate, generates PR metadata from the branch diff, and creates the GitHub PR with `gh pr create`.

**Architecture:** Keep the skill under `.codex/skills/create-pr/` with a concise `SKILL.md` and UI metadata in `agents/openai.yaml`. Match the existing repo-local skill layout and keep the implementation judgment-driven, with no helper scripts or reference files in the first version.

**Tech Stack:** Markdown, YAML, existing local skill-creator validation script if dependencies allow

---

### Task 1: Add the Skill Files

**Files:**
- Create: `.codex/skills/create-pr/SKILL.md`
- Create: `.codex/skills/create-pr/agents/openai.yaml`

- [ ] **Step 1: Write the skill frontmatter and overview**

Add `name: create-pr` and a trigger-only `description`, then describe the skill as an end-to-end PR creation workflow rather than a title-drafting helper.

- [ ] **Step 2: Write the main workflow**

Document the fixed order: inspect Git state, check `git config branch.<current-branch>.base`, infer the base branch if that override is absent, review uncommitted changes, create any needed commits, summarize `base...HEAD`, generate title/body, push, and call `gh pr create`.

- [ ] **Step 3: Add the decision rules**

Capture the approved rules for prioritizing `git config branch.<current-branch>.base`, fallback base-branch inference, uncommitted-change selection, commit splitting, stop conditions, and late `gh pr create` failure handling.

- [ ] **Step 4: Add the agent metadata**

Create `agents/openai.yaml` with `display_name`, `short_description`, and a `default_prompt` that explicitly mentions `$create-pr`.

### Task 2: Validate the Skill Packaging

**Files:**
- Review: `.codex/skills/create-pr/SKILL.md`
- Review: `.codex/skills/create-pr/agents/openai.yaml`

- [ ] **Step 1: Run the local validator if possible**

Run: `python3 .codex/skills/.system/skill-creator/scripts/quick_validate.py .codex/skills/create-pr`
Expected: Validation passes, or Python reports the missing `yaml` dependency so the limitation can be recorded.

- [ ] **Step 2: Perform manual structural review**

Check that the frontmatter contains only `name` and `description`, the description is trigger-oriented, the workflow matches the approved spec, the output contract is concise, and no extra resource directories were added.

- [ ] **Step 3: Perform a lightweight behavior review**

Manually verify that the written rules support these scenarios:
- clean branch with existing commits only
- clearly related uncommitted changes added before PR creation
- mixed related and unrelated local changes
- ambiguous base branch requiring a user question
- `gh pr create` failure after metadata generation
